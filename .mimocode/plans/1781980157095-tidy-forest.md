```markdown
# AGENTS.md

## Build & Development Commands
- Build: `npm run build` (TypeScript + frontend)
- Dev: `npm run dev`
- Test: `npm test`
- Lint: `npm run lint`
- Frontend build: `cd frontend && npm run build`

## Architecture
- ESM modules throughout (`"type": "module"`)
- TypeScript 5.4+ with strict mode, ES2022 target
- CLI (Commander.js) → Scanner (orchestrator) → 25 parsers → DependencyGraph
- Three interfaces: CLI, Web Dashboard (React+Express), MCP Server

## Code Style
- No comments unless requested
- ESM imports with `.js` extension
- Zod for schema validation
- PathResolver abstracts all filesystem operations

## Project Structure
- src/cli.ts - CLI entry point (5 commands)
- src/core/scanner.ts - Central orchestrator
- src/parsers/ - 25 config parsers
- src/mcp/ - MCP server (17 tools)
- src/server/ - Express API (14+ endpoints)
- frontend/ - React/Vite UI with @xyflow/react

## Testing
- Vitest with globals
- Tests in tests/ directory
- Create temp dirs in beforeEach, clean in afterEach
- Mock filesystem for parser tests

## Key Patterns
- Strategy pattern for parsers
- DependencyGraph with typed edges (depends, builds, network, volume, routes)
- MCP dual transport (stdio + HTTP)
```
