# local-architecturer — Audit & Optimization Dönüşümü

## Proje Özeti
Mevcut: pnpm monorepo + Docker analiz aracı  
Hedef: Tam teşekküllü mimari denetim ve optimizasyon sistemi

---

## Önerilen Geliştirmeler

### 1. Çoklu Ekosistem Desteği (Multi-Language Parsing)
**Mevcut Durum:** Sadece Node.js (pnpm-workspace.yaml / package.json) destekleniyor

**Eklenecekler:**
- `composer.json` parser (PHP/Laravel ekosistemi)
- `requirements.txt` / `pyproject.toml` parser (Python ekosistemi)
- `Cargo.toml` parser (Rust ekosistemi) — opsiyonel
- `go.mod` parser (Go ekosistemi) — opsiyonel

**Kullanım Senaryosu:** Next.js ön yüz + Laravel arka plan + Python ML servisi tek grafikte

**Dosyalar:**
- `src/parsers/composer-parser.ts` (yeni)
- `src/parsers/python-parser.ts` (yeni)
- `src/types/index.ts` — yeni paket türleri ekle
- `src/core/scanner.ts` — çoklu ekosistem tarama mantığı

---

### 2. Çevresel Değişken (.env) Kapsama Haritası
**Mevcut Durum:** Environment değişkenleri sadece docker-compose parsing'de okunuyor, eşleştirme yok

**Eklenecekler:**
- `.env.example` dosyası okuma
- Kaynak kodlarda `process.env.X`, `os.environ["X"]`, `getenv("X")` tarama
- docker-compose `environment` bloklarıyla eşleştirme
- Eksik değişken tespiti ve uyarı sistemi

**Çıktı:**
- CLI: `Warning: Service "api" uses DB_HOST but it's not defined in .env.example`
- MCP: `get_env_coverage` tool'u
- Dashboard: Eksik değişkenleri gösteren panel

**Dosyalar:**
- `src/parsers/env-parser.ts` (yeni)
- `src/core/env-analyzer.ts` (yeni)
- `src/mcp/server.ts` — yeni tool ekle

---

### 3. Döngüsel Bağımlılık Tespiti
**Mevcut Durum:** `get_health_score` tool'unda basit DFS var, ama CLI ve API'de yok

**Eklenecekler:**
- `arch-viz analyze --check-circular` komutu
- Grafikte döngüsel bağımlılıkları kırmızı ok ile gösterme
- CI pipeline entegrasyonu (exit code 1 ile hata fırlatma)
- Hangi paketlerin döngüde olduğunu detaylı listeleme

**Dosyalar:**
- `src/core/circular-detector.ts` — MCP server'dan çıkarılıp ortak modüle taşınmalı
- `src/commands/analyze.ts` — `--check-circular` flag'i
- `src/mcp/server.ts` — mevcut kodu refactor et
- `frontend/src/components/GraphView.tsx` — kırmızı edge render'ı

---

### 4. Docker Kaynak ve Güvenlik Denetimi
**Mevcut Durum:** Sadece port çakışmaları ve basit analiz var

**Eklenecekler:**
- `deploy.resources.limits` (RAM/CPU) kontrolü
- `healthcheck` tanımlama zorunluluğu
- `restart: always` / `unless-stopped` kontrolü
- `read_only: true` (güvenlik) önerisi
- GPU/VRAM tanımlama kontrolü (AI servisleri için)
- ` privileged: true` uyarıları

**Çıktı:**
- CLI: `Warning: Service "ml-inference" has no resource limits defined`
- MCP: `audit_docker_security` tool'u
- Dashboard: Güvenlik skoru paneli

**Dosyalar:**
- `src/core/docker-auditor.ts` (yeni)
- `src/mcp/server.ts` — yeni tool
- `src/mcp/port-conflict-detector.ts` — genişletme

---

### 5. Yerel LLM "Review" Komutu
**Mevcut Durum:** MCP sunucusu var ama CLI'dan doğrudan AI inceleme yok

**Eklenecekler:**
- `arch-viz review` komutu
- Ollama / LM Studio / OpenRouter entegrasyonu
- Graph JSON'u AI'a gönderme
- Mimari tavsiyeleri terminal'e yazdırma

**Kullanım:**
```bash
arch-viz review --provider ollama --model qwen2.5
arch-viz review --provider openrouter --model deepseek/deepseek-r1
```

**Çıktı Örneği:**
```
[Review] Service "db" has no persistent volume defined
[Review] Package "ui" has too many circular dependencies
[Review] Consider adding healthcheck to "api" service
```

**Dosyalar:**
- `src/commands/review.ts` (yeni)
- `src/ai/review-engine.ts` (yeni)
- `src/cli.ts` — yeni komut register

## Doğrulama

1. `npm run lint` — ESLint temiz
2. `npm run typecheck` — TypeScript compile başarılı
3. `npm test` — Tüm testler geçmeli
4. `npm run build` — Build başarılı
5. Manuel test:
   - `arch-viz analyze . --check-circular`
   - `arch-viz analyze .` (env uyarılarını kontrol et)
   - `arch-viz serve` (dashboard'da yeni panel'leri kontrol et)
   - MCP tool'larını test et (Claude Desktop veya benzeri)
