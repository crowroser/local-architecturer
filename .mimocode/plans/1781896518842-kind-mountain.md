# local-architecturer Geliştirme Planı

## Mevcut Durum
- **Versiyon**: 0.1.0
- **Durum**: Temel özellikler çalışıyor (analyze, serve, MCP server)
- **Eksikler**: Aşağıda detaylandırılmıştır

---

## Öncelik 1: Kritik Hatalar (Hemen Düzeltmeli)

### 1.1 `cors` bağımlılık eksik
- `src/server/express-server.ts` cors import ediyor ama `cors` package.json'da runtime dependency olarak yok
- Sadece `@types/cors` devDependencies'te var
- **Çözüm**: `cors` paketini dependencies'e ekle

### 1.2 Frontend build entegrasyonu
- `dist/public/` klasöründe built dosyaları var ama build scripti sadece `tsc` çalıştırıyor
- Frontend build'i ayrı çalıştırılıyor, ana build scriptine eklenmeli
- **Çözüm**: `build` scriptini güncelle: `"build": "tsc && cd frontend && npm run build"`

---

## Öncelik 2: Yeni Özellikler (Feature Requests)

### 2.1 Çoklu Package Manager Desteği
**Şu an**: Sadece pnpm workspace desteği var
**Önerilen**: npm workspaces, yarn workspaces desteği
- `workspace-parser.ts`'e npm/yarn formatı ekle
- `package.json`'daki `workspaces` alanını oku (npm/yarn için)

### 2.2 Dockerfile İçerik Analizi
**Şu an**: Dockerfile'lar sadece tespit ediliyor, içeriği analiz edilmiyor
**Önerilen**: 
- Base image bilgisini çıkar
- EXPOSE portlarını parse et
- Multi-stage build tespiti
- Build arg'ları tespit

### 2.3 MCP HTTP Transport
**Şu an**: Sadece stdio transport çalışıyor
**Önerilen**: Streamable HTTP transport ekle
- AI asistanları uzaktan bağlanabilsin
- `src/commands/mcp.ts`'de HTTP seçeneği ekle

### 2.4 Lockfile Analizi
**Önerilen**: pnpm-lock.yaml analizi
- Tam bağımlılık ağacını çıkar
- Versiyon sabitleme durumunu göster
- Güvenlik açığı tarama entegrasyonu (npm audit)

### 2.5 Proje Sağlık Skoru
**Önerilen**: Analiz sonucunda sağlık skoru hesapla
- Bağımlılık karmaşıklığı skoru
- Güvenlik riski skoru
- Performans riski skoru (büyük monorepo'lar için)

## Öncelik 3: Frontend Geliştirmeleri

### 3.1 Graph Filtreleme
- Package/service türüne göre filtre
- Arama özelliği (paket adı ile)
- Bağımlılık zincirini vurgulama

### 3.2 Detay Paneli
- Node tıklandığında sağ panel açılsın
- Package detayları: version, dependencies, devDependencies
- Service detayları: image, ports, volumes, environment

### 3.3 Zoom ve Navigasyon
- Mini map ekle
- Belirli bir node'a odaklanma
- Zoom in/out kontrolleri

### 3.4 Tema Desteği
- Dark/Light mode
- Kullanıcı tercihlerini kaydet

## Öncelik 4: Altyapı Geliştirmeleri

### 4.1 CI/CD Pipeline
- GitHub Actions workflow ekle

### 4.3 Hata Yönetimi
- Parser'larda robust hata yakalama

- Debug modu için detaylı loglama

### 4.4 Favicon ve PWA
- Favicon ekle (frontend/index.html referans veriyor)
- PWA manifest ekle (opsiyonel)
